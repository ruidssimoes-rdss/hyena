## Design Tokens

r/ui uses a token-based system for consistent styling. All values are defined in `@r-ui/react-native/tokens`.

### Colors

```ts
import { colors } from '@r-ui/react-native'

// Background hierarchy (5 levels)
colors.bg.base      // '#050505' — App background
colors.bg.raised    // '#0a0a0a' — Slightly elevated
colors.bg.surface   // '#141414' — Cards, inputs
colors.bg.elevated  // '#1f1f1f' — Modals, popovers
colors.bg.overlay   // 'rgba(0,0,0,0.8)' — Backdrop

// Text hierarchy
colors.text.primary   // '#ffffff' — Main text
colors.text.secondary // 'rgba(255,255,255,0.6)' — Secondary
colors.text.muted     // 'rgba(255,255,255,0.4)' — Hints
colors.text.inverse   // '#0a0a0a' — On light backgrounds

// Borders
colors.border.default // 'rgba(255,255,255,0.1)'
colors.border.muted   // 'rgba(255,255,255,0.06)'
colors.border.strong  // 'rgba(255,255,255,0.2)'

// Accent colors (each with light/DEFAULT/dark)
colors.accent.blue.light   // '#60a5fa'
colors.accent.blue.DEFAULT // '#3b82f6'
colors.accent.blue.dark    // '#2563eb'

colors.accent.green.DEFAULT  // '#22c55e'
colors.accent.amber.DEFAULT  // '#f59e0b'
colors.accent.red.DEFAULT    // '#ef4444'
colors.accent.purple.DEFAULT // '#a855f7'

// Semantic colors
colors.semantic.success // '#22c55e'
colors.semantic.warning // '#f59e0b'
colors.semantic.error   // '#ef4444'
colors.semantic.info    // '#3b82f6'
```

## Using Tokens

### Direct Import

```tsx
import { colors, spacing, radius } from '@r-ui/react-native'
import { View, Text, StyleSheet } from 'react-native'

const styles = StyleSheet.create({
  card: {
    backgroundColor: colors.bg.surface,
    borderRadius: radius.lg,
    padding: spacing[4],
    borderWidth: 1,
    borderColor: colors.border.default,
  },
  title: {
    color: colors.text.primary,
    fontSize: fontSizes.lg,
    marginBottom: spacing[2],
  },
})
```

### With NativeWind

If using NativeWind, tokens are available as Tailwind classes:

```tsx
<View className="bg-surface rounded-lg p-4 border border-default">
  <Text className="text-primary text-lg mb-2">Title</Text>
</View>
```

## CSS Variables (Web)

On web, r/ui uses CSS variables for theming. This enables smooth theme transitions and runtime customization.

```css
:root {
  --color-bg-base: #050505;
  --color-bg-surface: #141414;
  --color-text-primary: #ffffff;
  --color-accent-blue: #3b82f6;
  /* ... */
}

[data-theme="light"] {
  --color-bg-base: #ffffff;
  --color-bg-surface: #f5f5f5;
  --color-text-primary: #0a0a0a;
  /* ... */
}
```

## Customizing Tokens

### Override in Your Theme

Create a custom theme by overriding CSS variables:

```css
/* globals.css */
:root {
  /* Override accent color */
  --color-accent-blue: #6366f1; /* Indigo instead of blue */

  /* Override radius for sharper corners */
  --radius-md: 4px;
  --radius-lg: 8px;
}
```

### Tailwind Config

Extend your `tailwind.config.js`:

```js
module.exports = {
  theme: {
    extend: {
      colors: {
        // Add custom colors
        brand: {
          50: '#f0f9ff',
          500: '#0ea5e9',
          900: '#0c4a6e',
        },
      },
      spacing: {
        // Add custom spacing
        '18': '4.5rem',
      },
    },
  },
}
```

## Creating a Custom Theme

### Step 1: Define Your Palette

```ts
// theme/custom.ts
export const customColors = {
  bg: {
    base: '#1a1625',      // Deep purple-black
    raised: '#231f2e',
    surface: '#2d2640',
    elevated: '#3d3556',
    overlay: 'rgba(26, 22, 37, 0.9)',
  },
  text: {
    primary: '#f8f4ff',
    secondary: 'rgba(248, 244, 255, 0.7)',
    muted: 'rgba(248, 244, 255, 0.4)',
  },
  accent: {
    primary: '#a78bfa', // Purple
  },
}
```

### Step 2: Create CSS Variables

```css
[data-theme="custom"] {
  --color-bg-base: #1a1625;
  --color-bg-raised: #231f2e;
  --color-bg-surface: #2d2640;
  --color-bg-elevated: #3d3556;
  --color-text-primary: #f8f4ff;
  --color-accent-blue: #a78bfa;
  /* ... */
}
```

### Step 3: Use the Theme

```tsx
import { ThemeProvider, useTheme } from '@r-ui/react-native'

// Wrap your app
function App() {
  return (
    <ThemeProvider defaultTheme="dark">
      <MyApp />
    </ThemeProvider>
  )
}

// Toggle theme in a component
function ThemeToggle() {
  const { theme, setTheme } = useTheme()

  return (
    <View style={{ flexDirection: 'row', gap: 8 }}>
      <Button
        variant={theme === 'dark' ? 'primary' : 'ghost'}
        onPress={() => setTheme('dark')}
      >
        Dark
      </Button>
      <Button
        variant={theme === 'light' ? 'primary' : 'ghost'}
        onPress={() => setTheme('light')}
      >
        Light
      </Button>
      <Button
        variant={theme === 'oatmeal' ? 'primary' : 'ghost'}
        onPress={() => setTheme('oatmeal')}
      >
        Oatmeal
      </Button>
    </View>
  )
}
```

## Glass Theme

The Glass theme adds a frosted glass morphism effect to components, creating translucent surfaces with backdrop blur. This is a visual enhancement that works alongside the base theme (Dark, Light, or Oatmeal).

### Enabling Glass Mode

```tsx
import { ThemeProvider } from '@r-ui/react-native'

function App() {
  return (
    <ThemeProvider defaultTheme="dark" isGlass>
      {/* Components will render with glass effect */}
    </ThemeProvider>
  )
}
```

### How Glass Mode Works

When glass mode is enabled:
- Components use the `GlassSurface` wrapper for backdrop blur effects
- Backgrounds become translucent with subtle white overlay
- Borders use translucent white (`rgba(255,255,255,0.2)`)
- Shadows are enhanced for depth on glass surfaces
- Text colors may adjust for optimal contrast

### GlassSurface Component

The core of the glass effect is the `GlassSurface` component:

```tsx
import { GlassSurface } from '@r-ui/react-native'

<GlassSurface
  intensity={16}        // Blur intensity (0-100)
  opacity={0.7}         // Background opacity
  borderRadius={14}     // Corner radius
  shadow="md"           // Shadow size: 'none' | 'sm' | 'md' | 'lg'
  bordered              // Add translucent border
>
  <Text>Content on glass surface</Text>
</GlassSurface>
```

### Glass-Enabled Components

22 components support the glass theme:

**Surfaces & Containers:**
- Card, Dialog, Sheet, Toast, Popover, EmptyState

**Menus & Selection:**
- Dropdown, ContextMenu, Select, Command, NavigationMenu, ActionSheet

**Interactive Elements:**
- AlertDialog, Tooltip, HoverCard, Accordion, Collapsible, Tabs

**Layout & Navigation:**
- Onboarding, Announcement, Sidebar, Navbar

### Glass Design Tokens

| Property | Value | Description |
|----------|-------|-------------|
| `intensity` | `12-20` | Blur amount (higher = more blur) |
| `opacity` | `0.5-0.7` | Background transparency |
| `borderRadius` | Follows `radius.*` tokens | Corner rounding |
| `shadow` | `sm` to `lg` | Shadow depth |
| `borderColor` | `rgba(255,255,255,0.2)` | Subtle glass border |

### Recommended Backgrounds

Glass effects work best over colorful or textured backgrounds:

```tsx
// Warm gradient background for glass surfaces
<View style={{
  flex: 1,
  background: 'linear-gradient(135deg, #E8E4DF 0%, #D4CFC7 100%)',
}}>
  <Card>
    {/* Card renders with glass effect */}
  </Card>
</View>
```

### Checking Glass Mode in Custom Components

Access the glass state in your own components:

```tsx
import { useTheme } from '@r-ui/react-native'

function MyComponent() {
  const { isGlass } = useTheme()

  return isGlass ? (
    <GlassSurface intensity={16} borderRadius={14}>
      <Text>Glass mode content</Text>
    </GlassSurface>
  ) : (
    <View style={{ backgroundColor: colors.bg.surface }}>
      <Text>Default mode content</Text>
    </View>
  )
}
```

### Performance Considerations

- Backdrop blur uses `backdrop-filter` on web (CSS) and `BlurView` on native
- Glass effects can impact performance on lower-end devices
- Consider providing a non-glass fallback for accessibility
- The `useReducedMotion` hook can be used to disable glass effects when reduced motion is preferred
